XML:SetFieldPicture

Parameters:

STRING

Field Name


STRING

A valid Clarion picture to format the string before writing
 
If the picture is * (an asterisk), then the field is written exactly as-is without any XML/HTML conversion.  This is helpful if you want to write the field out for HTML files and do not want a conversion to happen to the field.
 
Wild Cards are supported such as *DATE* to change all date fields.



Return Type:

Does not return a value

 

Instructs the writer to format all data being written out, either manually or by XML:AddFromFile, XML:AddFromQueue, XML:AddFromGroup.  The picture must be a valid Clarion Picture Token such as @d010-

 

To manually clear any set field pictures, call this function with a blank picture to release the field.  However, see the following:

 

NOTE:  All Field Pictures are freed when called XML:CloseXMLFILE() or XML:CloseXMLString() is called.

 

 

Example:

 

 ! convert CLARION long field to YYYY-MM-DD and format money

 ! ---------------------------------------------------------

 XML:SetFieldPicture('InvoiceTotal','@n-12.2')

 XML:SetFieldPicture('InvoiceDate','@d010-')                

 

 ! Do some processing

 

 XML:SetFieldPicture('InvoiceTotal','')

 XML:SetFieldPicture('InvoiceDate','')

 

 

 ! Do not internally convert fields to value XML

 ! ---------------------------------------------------------

 XML:SetFieldPicture('td','*')

 

 xml:CreateParent('td')

 xml:AddParent(True,'&nbsp;hello&nbsp;'

 

 XML:SetFieldPicture('td','')

 

 In the above example, the field will be written as &nbsp; instead of &amp;nbsp;

 

 

 ! convert all fields to date where date is in the field name

 ! ---------------------------------------------------------

 XML:SetFieldPicture('*date*','@d010-')                

 

 ! Do some processing

 

 XML:SetFieldPicture('*date*','')
