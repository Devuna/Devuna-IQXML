Write - AddFromQueue Example

The following code will create an XML document from the contents of MyQueue.  Group Fields will create child records.  The dimensioned Invoice Group will create a record for each dimension.   The local program can easily fill MyQueue wil

 

 

I

It is HIGHLY RECOMMENDED to only use STRING fields in your Queue structure, especially when processing child records.

 

The system has to determine when to create the next Queue entry properly, and comparing a BLANK value in an XML file is different than comparing a "0" in the Queue str

 

 

 

Example 1 (Groups inside of Groups):

 

MyQueue

QUEUE,PRE()                

Lastname

STRING(20)              

Firstname

STRING(20)              

Address

GROUP,PRE()

Streets

GROUP,PRE()                

StreetName

STRING(35),DIM(3)      

END                        


City

STRING(20)            

State

STRING(20)                  

Zipcode

STRING(20)                  

Country

STRING(20)                  

END                        


Invoices

GROUP,PRE()                

Invoice

GROUP,PRE()                

InvoiceAmt

STRING(15),DIM(100)        

InvoiceDate

STRING(10),DIM(100)        

InvoiceNo

STRING(10),DIM(100)        

END

END

END

 

If ~XML:CreateXMLFile('MyFile','1.0')

 XML:CreateParent('XMLData');XML:AddParent())

 XML:AddFromQueue(MyQueue,'Customer',True)

 XML:CloseParent()

 XML:CloseXMLFile()

End

 

<?xml Version="1.0"?>

<XMLData<Customer<ADDRESS<STREETS<CITY</ADDRESS<INVOICES<INVOICE<INVOICEAMT<INVOICEDATE<INVOICENO</INVOICE<INVOICE<INVOICEAMT<INVOICEDATE<INVOICENO</INVOICE</INVOICES</Customer</XMLData
