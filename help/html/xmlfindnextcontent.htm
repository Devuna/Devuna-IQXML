XML:FindNextContent

Parameters:

STRING

Content text string

BYTE

Exact Match Type (True = Exact Match without Instring)

BYTE

Exact Case Type (True = Exact Case Match)



Return Type:

SHORT

XML Error Code (see end of document)

 

Start from the current node pointer and look for content that matches the value passed. Use can then use the XML:ReadCurrentRecord function if the function returns successful to determine the exact value and node returned if needed.

 

This function will only search for a node pointer beneath the current parent.  If there is no node pointer set or current record set, the function will start at the top of the XML document.

 

Example:        

 

! Will find first content of "Hello" and "Hello There"

! ----------------------------------------------------

IF NOT XML:FindNextContent('hello',False,False)        

 IF NOT  XML:ReadCurrentRecord(XMLQUEUE,CHECKVALUE)

         Message('The real value is ' & checkvalue)

 

 

! Will find first content of "Hello" only

! ----------------------------------------------------

IF NOT XML:FindNextContent('Hello',True,True)        

 If not XML:ReadCurrentRecord(XMLQUEUE,CheckValue)
