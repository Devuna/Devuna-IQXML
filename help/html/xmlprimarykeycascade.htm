XML:PrimaryKeyCascade

Parameters:

STRING

Field to cascade to new queue fields

<SHORT>

IQXML:CASCADE_DEFAULT

IQXML:CASCADE_PARENT

IQXML:CASCADE_SAVE
 



Return Type:

Does not return a value

 

Used to tell the XML:LoadQueue function to not clear these fields when adding new 'child' entries to the queue.

 

If you are using XML:LoadQueue to add child entries to a local queue, you may want to keep certain fields primed for each entry in to the Queue. This allows your local program to read queue entries and know which records belong to which parent.  These values are maintained through-out the life of your thread until XML:PrimaryKeyClear is called.

 

IQXML:CASCADE_DEFAULT (0) - When the Node Pointer moves on to the next sibling element, the field is cleared.  This value will remain primed for the length of all nodes and inserted records for the related sibling.

 

IQXML:CASCADE_PARENT (1) - When the Node Pointer moves back to a parent node for which the field is first found, the field is cleared.  This value will remain primed for the length of its child nodes and all inserted records for the related child.

 

IQXML:CASCADE_SAVE (2) - Once this field is encountered and primed, it will stay primed for the entire length of the XML:LoadQueue() call, including Parents, Siblings and Children for all inserted records.

 

 

 

See also XML:PrimaryKeyClear, XML:QualifyFieldSet, XML:PrimaryFieldCascade

 

 

Example:

 

<Customers>

<Customer CustomerID=123 CustomerName="ACME Electric">

 <Trucking Method="Best Way" Carrier="ABC Freight"/>

  <Invoices>

<Invoice InvoiceNumber=1 InvoiceDate="01/01/2005" InvoiceAmount="123.45"/>

<Invoice InvoiceNumber=2 InvoiceDate="01/02/2005" InvoiceAmount="345.50"/>

<Invoice InvoiceNumber=3 InvoiceDate="01/02/2005" InvoiceAmount="11.75"/>

</Invoices>

</Customer>

<Customer CustomerID=124 CustomerName="Ace Electric">

  <Invoices>

<Invoice InvoiceNumber=4 InvoiceDate="01/01/2005" InvoiceAmount="599.00"/>

<Invoice InvoiceNumber=5 InvoiceDate="01/01/2005" InvoiceAmount="12.00"/>

<Invoice InvoiceNumber=6 InvoiceDate="01/01/2005" InvoiceAmount="16.75"/>

</Invoices>

</Customer>

</Customers>

! ------------------------------------------------------------------------------

! Read XML Document which contains a header entry of CustomerID and CustomerName

! and a group of detail records.  Attach all the header information with

! each detail record

 

! The above XML file will go into the above queue with the following

! four lines of code, and the Customer Number and ID will be populated

! in each queue entry.  When the program finishes, there will be

! 6 records in MyQueue.

!

! ** In this example, the Trucking Methods and Carriers will appear on all six

! of the invoices.  If the CASCADE_SAVE value were changed to CASCADE_DEFAULT,

! then the trucking method would only appear on Customer ID 123.

! ------------------------------------------------------------------------------

 

XML:QualifyFieldSet('Method')

XML:QualifyFieldSet('Carrier')
 

XML:PrimaryKeyCascade('CustomerID',IQXML:CASCADE_DEFAULT)

XML:PrimaryKeyCascade('CustomerName',IQXML:CASCADE_DEFAULT)
 

XML:PrimaryKeyCascade('TruckingMethod',IQXML:CASCADE_SAVE)

XML:PrimaryKeyCascade('TruckingCarrier','IQXML:CASCADE_SAVE)
 

If NOT XML:FindNextNode('Customer')

 AddedRecords = XML:LoadQueue(MyQueue,True,True)

 XML:PrimaryKeyClear()

end
