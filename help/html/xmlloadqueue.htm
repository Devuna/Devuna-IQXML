XML:LoadQueue

Parameters:

*QUEUE

Local queue to catch values

<BYTE False>

True to process all Child nodes


<BYTE False>

True to process all Sibling nodes

<BYTE False>

Include field prefixes when matching field names such as QUE:fieldname.  Only the first prefix is ignored if there are multiple prefixes such as PRE:PRE:ActualField.
 
0 = Ignore prefixes in Clarion local queue

1 = Include any prefixes in Clarion local queue

2 = Ignore prefixes in Clarion local queue and any prefix/namespaces in XML document
 

<LONG>

Maximum number of records to write to Queue before Aborting.

<BYTE>

True to Ignore Queue Field usage checking and only write during IQXML:FORCEWRITE




Return Type:

LONG

Records added to the Queue

 

Used to retrieve values from the loaded XML file and store them in the Clarion queue with matching field names.

 

For each group within the set node at the current cursor location, a record will be added to the queue.  A queue can contain dimensional array fields.  If a field in the queue is found more than once (when looking in child nodes and/or processing all sibling nodes), a new record will be added to the queue.  The exception to this is if the field found is declared as an array.    You can use the XML:AttributeFieldSet function to further define when to write a record to the Queue and what fields are required or optional.

 

Once you are finished using a queue, you should issue XML:Free() to clear the memory.  Failure to call XML:Free() can cause memory corruption.

 

** It is HIGHLY RECOMMENDED to only use STRING fields in your Queue structure, especially when processing child records.  The system has to determine when to create the next Queue entry properly, and comparing a BLANK value in an XML file is different than comparing a "0" in the Queue structure.  The same would be for Decimal fields when the XML file may contain "" but the Queue structure would contain "0" or "0.00"

 

 

NOTE: DIMMED Field and Multi-Group Processing is not valid with Clarion 5.5 Version of iQ-XML

 

 

When this function completes:
 

The node pointer is changed when executing this function which will be the end of file or start of next section.  To keep the node pointer at the same location before it executed, use XML:AutoRootSet.  

 

 

When Queue Entries get added to the Queue:

 

1.When a field value is needed to be placed in the queue, and the queue's current field value is non-blank.
2.When the XML element has moved on to a parent node.
3.When the end of the XML document is read and at least one of the fields in the Queue was filled in.
4.When a queue field has been filled in and that field has been set with the IQXML:FORCEWRITE flag

 

 

Ignore Queue Field Usage and only write during IQXML:FORCEWRITE

 

iQ-XML figures out when to add a field to the queue based on the above four reasons.  However, if you are using non-string fields (LONG, BYTE, DATE, DECIMAL, etc.), then item #1 above would always come into place for every single field causing a record to be written for every field in the queue.

 

To avoid the auto checking of field usage (ie: non blank adding of a queue record), designate the field you wish to trigger the queue add using the XML:AttributeFieldSet() function with the IQXML:FORCEWRITE indicator and set the "Ignore Queue Field usage checking" parameter to True (1).  However, be warned that if a field has already been filled in, it would just be overwritten if not properly written out.  Always be sure this one field you indicate is the last field in the record.

 

For example, you would use the following when working with the following data:

 

<Invoice Invoiceno="1" InvoiceDate="2005-01-01" InvoiceAmt="12.12"/>

 

XML:AttributeFieldSet('InvoiceAmt',IQXML:FORCEWRITE)

Recs = XML:LoadQueue(MyQueue,True,True,false,,True)

 

In the above example, regardless of whether other fields have values, when the XML Reader hits the "InvoiceAmt" element in the XML document, the record is written to the MyQueue queue.  Please note that if MyQueue contained string fields for "InvoiceNo" and "InvoiceAmt", then this would not be needed.

Child/Sibling Processing Example:

 

 

 <root>

       <root2>

            <root3/>

          <root4/>

 </root2>

       <root2>

            <root3/>

          <root4/>

 </root2>

 </root>

 

 Say the pointer is on the first <root2> ...

 

 xml:LoadQueue(queue,True,False)        ! will process root2,root3,root4

 xml:LoadQueue(queue,True,True)        ! will process root2,root3,root4,root2,root3,root4

 xml:LoadQueue(queue,False,True)        ! will process root2,root2

 xml:LoadQueue(queue,False,False)        ! will process root2

 

 

 

See also XML:AttributeFieldSet, XML:AutoRootSet, XML:PrimaryKeyCascade, XML:PrimaryFieldCascade, XML:QualifyFieldSet

Mapping from an XML file with different field names:

 

Use the NAME attribute in Clarion to map element field names from the XML file that are declared one way, but you wish to declare another way in your program's queue.  For example, this is needed when field names in an XML document are reserved words in the Clarion language or invalid characters.

 

<root>

  <Data Member='iqxml' Code='23 lines' Actual-Name='something' Invalid:name='another thing' Field3='Robert' ActualField='And one more thing'>

  <Data Member='iqxml2' Code='6 lines' Actual-Name='something' Invalid:name='another thing' Field3='Robert' ActualField='And one more thing'>

</root>

STRING(20),NAME('Actual-Name')

Field2

STRING(20),NAME('Invalid:name')

STRING(20)

Mapping with non-string fields in a queue:

 

If you use numeric fields in your queue, you must use the XML:SetFieldAttribute() function and indicate non-automatic checking of field usage and that you will be controlling the write operation yourself with the IQXML:FORCEWRITE indicator.

 

 

MyQueue


QUEUE

CustNumber

LONG

Firstname

STRING(25)

Lastname

STRING(25)

AGE

BYTE

SEX

STRING(1)

PHONENUMBER

STRING(20)

 END

Recs

SHORT

 

 ! First, write the queue out to an XML file

 ! -----------------------------------------

 IF NOT XML:CreateXMLFile('MyFile.xml')

             XML:CreateParent('Customers');XML:AddParent()

               XML:AddFromQueue('Data',True,True)

         XML:CloseParent()

         XML:CloseXMLFile()

 

         ! Empty the Queue

         ! ---------------

         FREE(MyQueue)

 

         ! Now, read it back

         ! -----------------

         if NOT XML:LoadXMLFile('MyFile.Xml')

                 IF NOT XML:FindNextNode('Data')

                         XML:AttributeFieldSet('PHONENUMBER',IQXML:FORCEWRITE)

                         Recs = XML:LoadQueue(MyQueue,True,True,False,,True)

                 END

         END

 END

 

 ! Without the above parameter, and because numeric fields

 ! exist (AGE and CustNumber) you are indicating to only write

 ! the Queue entry when the PHONENUMBER field is encountered

 ! which is always the last field for each data record.
