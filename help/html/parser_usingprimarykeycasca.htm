Parser - Using PrimaryKeyCascade

If you are processing an XML document with both Child and Sibling parameters, you have the ability to keep certain fields primed so your program can read the data easily.

 

Take for instance the following xml data:

 

<xmlmessage>

<Customers>

<Customer CustomerName="Smith Travel" CustomerID=123>

  <Invoices>

    <Invoice Invoiceno="1" InvoiceDate="2005-01-01" InvoiceAmt="12.12"/>

    <Invoice Invoiceno="12" InvoiceDate="2005-03-02" InvoiceAmt="9.09"/>

    <Invoice Invoiceno="144" InvoiceDate="2005-04-02" InvoiceAmt="1000.00"/>

  </Invoices>

</Customer>

<Customer CustomerName="ABC Company" CustomerID=188>

  <Invoices>

    <Invoice Invoiceno="20" InvoiceDate="2005-01-01" InvoiceAmt="50.00"/>

    <Invoice Invoiceno="21" InvoiceDate="2005-02-01" InvoiceAmt="50.00"/>

    <Invoice Invoiceno="22" InvoiceDate="2005-03-01" InvoiceAmt="50.00"/>

    <Invoice Invoiceno="23" InvoiceDate="2005-04-01" InvoiceAmt="60.00"/>

  </Invoices>

</Customer>

</Customers>

</xmlmessage>

 

 

The following code will allow you to process this XML document easily in to one queue instead of two different queues.

 

 

InvoicesQueue

QUEUE

CustomerName

string(20)

CustomerID

string(20)

InvoiceNo

string(20)

InvoiceDate

string(20)

InvoiceAmt

string(20)

End


IRecs

SHORT

 

CODE

 

! Load the XML file to process it

! -------------------------------

If ~XML:LoadFromFile('testing.xml')

 If ~XML:FindNextNode('Customers','Customer')

 

         XML:PrimaryKeyCascade('CustomerName')

         XML:PrimaryKeyCascade('CustomerID')

         IRecs = XML:LoadQueue(InvoicesQueue,True,True)

 

 End

 XML:Free()

End

 

"IRecs" will contain the value of 7 (one for each invoice) keeping the CustomerName and Customer Number fields primed for all child records.
