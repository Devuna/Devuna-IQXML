DebugQueue Overview

The XML:DebugQueue function is a very helpful utility which allows you to see the data that iQ-XML is working with.   Debugging does not corrupt or change the current record or Node Pointer values when returning back to your application.

 

 F3 – Finds a value as an element, attribute or data.

 F4 – Sets the current Node Pointer and calls XML:FindNextNode()

 F5 – Go to Parent

 F6 – Go to Sibling

 F7 – Go to Child

 F10 - Expand/Collapse Detail Data

 

Copy Element to Clipboard – Puts the current selected record on the Windows clipboard as a XML:FindNextNode() parameter call to be easily pasted in your Clarion application.

 

Create Queue – Takes all marked records (tagged with either the + or – icons in the bottom right-hand corner) in the debug queue and creates a Clarion Queue structure to be easily pasted in your Clarion application when you want to use the XML:LoadQueue() function.  Simply run the Example application and open your XML file from the File menu.  Tag each field name you want imported in to your Clarion application, and then press the Create Queue button.  Once the queue structure is created, simply paste the contents of the clipboard in to your Data section of your application.

 

You can also click on any column header to find all unique values in that column and reposition the debug queue to the closest matching value.
