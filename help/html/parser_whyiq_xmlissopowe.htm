What I told you that you can pull all the names, address, multiple phone numbers from this document in two lines of code?  In addition, one set of these addresses are for the Customer and the other is for a Travel Agent, and you'll still be able t

 

Ready?  Here they are ...

 

XML:FindNextNode('Profiles','ProfileInfo','Profile)

Recs = XML:LoadQueue(ResQueue,True,True)

 

Yup, that's it.  I've successfully pulled out the names, addresses and phone numbers from this XML document.

 

It's all in how the QUEUE structure is setup.

When completed, this QUEUE will contain two records.  One for ProfileType = 1 and one for ProfileType = 2 - which is a "Parent Element" located in this document.

Our QUEUE would have 200 records in them, all with a cascaded unique identifier for every reservation. As you can see, processing XML documents are easy with iQ-XML.  You can't do this with Clarion XML functions.
