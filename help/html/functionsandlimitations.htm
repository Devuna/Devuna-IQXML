Functionality and Limitations

Currently, this parser and writer have the following functions and limitations; however, if there is an issue with any of them, we can release updates to expand the functionality as requested. iQ-XML is currently a non-validating parser.  If there is a missing tag during the "Load" functions, an error will display in a message box (unless Silent Mode is selected) and the function will return an error code.

Unlimited

Unlimited

Max Length for Binary Files

Unlimited

Unlimted for Parsing and Writing

 

iQ-XML can handle both CDATA and Comments.  Comments are completely ignored by the parser.  On the other-hand, CDATA is passed back to the application in its complete form without any modification, up to 10K (10240 bytes) per element.   As with standard "XML Version 1.0" rules, tag names are case-sensitive and must match exactly.

 

Values are also supported and automatically changed when parsing and writing such as &amp; &apos; &lt; &gt; &quot;
