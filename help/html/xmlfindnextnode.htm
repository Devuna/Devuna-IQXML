XML:FindNextNode

Parameters:

STRING

Element Name

<STRING>

Optional Element Name 2

<STRING>

Optional Element Name 3

<STRING>

Optional Element Name 4

<STRING>

Optional Element Name 5

<BYTE>

Implicit Element (True)



Return Type:

SHORT

XML Error Code (see end of document)

 

Sets the current Node Pointer to the next match of element(s) passed, setting the new parent.  Using this function will always start at the current cursor location.  If a "next node" is found, the cursor location is updated to the new position.  If the "next node" is not found, the cursor will not change.

 

Attributes are valid when using this function, however, you are not able to use another XML:FindNextNode to get another attribute within the node as this function only searches within nodes.  Finding a node with a prefix must include the prefix such as xml:FindNextNode('xsd:documentation') or you can use a WildCard (*) in position 1 to indicate an "INSTRING" search rather than implicit search.

 

 

Implicit Element - Set this optional flag to true (1) to indicate that the node to find is implicitly this Element/Node, and no other child or sub-node.  This is simply used to find the value of a node that also has attributes in it as well.

 

<messageheader systemid="myid" logon="robert">This is the message header info</messageheader>

 

If you use the following, the pointer will end up at the beginning of the message header:

 

xml:FindNextNode('Messageheader')

 

If you use the following, the pointer will end up at the actual value.  If no value, the pointer will still go to that location, and the value would be blank.

 

 xml:FindNextNode('Messageheader',,,,,True)

 

Wild Card (Name Space) searching:

 

 

By placing a * (asterisk) in position 1 of any of the parameters, a search for the node name will not be exact/implicit.   This is helpful if the XML is using namespaces and you are not aware of what the namespace may be.   For example, if a Soap Envelope is sending two documents inside it, with name spaces, you can still find the node by using a wildcard.

 

 

<soap:envelope>

<soap:header/>

  <soap:body>

    <doc1:root>

    <doc1:name first="Robert" last="Paresi"/>

    </doc1:root>

    <doc2:root>

    <doc2:name first="Robert" last="Paresi"/>

    </doc2:root>

</soap:body>

</soap:envelope>

 

if not xml:FindNextNode('*name')

 if not xml:FindNextNode('*name')

 

 

Above will find doc1:name and then doc2:name --- be aware that it does not break apart the NameSpace vs the Tag name, it is simply using a wildcard search for the word "name".

 

 

See also: XML:DebugTrace

 

Example:

 

<messageheader systemid="myid" logon="robert">

 

You can find "messageheader,systemid" and it will position the cursor on this attribute.  However, you cannot then issue another call to this function for "logon", as it will not work because the node that should have been searched for was "messageheader", which is the actual parent.  To retrieve the values of multiple attributes, you should issue the FindNextNode to the parent (messageheader) and then use the XML:LoadQueue to read all the values needed, or use the XML:ReadCurrentRecord/XML:ReadNextRecord functions checking the pointer location values.

 

This function will only find a new node pointer after the current node pointer location and within the current parent. If there is no node pointer set or current record set, the function will start at the top of the XML document.  To start searching from the top of document, you must call the XML:GotoTop function prior to calling this function, or use the xml:GotoParent / XML:GotoSibling function to continue searching further on in the document if needed.

 

<xmlmessage>

<messageheader systemid="myid" logon="robert">

  <message_type>Document 1</message_type>

    <datetime>0001-01-01T00:00:00</datetime>

</messageheader>

<Customers>

<Customer>

  <PersonName>

<GivenName>Robert</GivenName><Surname>Paresi</Surname></PersonName>

  <Address>

      <AddressLine>111 Main Street</AddressLine>

      <AddressLine>PO Box 12</AddressLine>

  </Address>

  <CityName>Hartford</CityName>

  <PostalCode>06331</PostalCode><StateProv>CT</StateProv>

</Customer>

<Customer>

  <PersonName>

<GivenName>Robert</GivenName><Surname>Jones</Surname></PersonName>

  <Address>

      <AddressLine>1 Dunninghurst Ave</AddressLine>

  </Address>

  <CityName>Elmhurst</CityName>

  <PostalCode>12122</PostalCode><StateProv>NY</StateProv>

</Customer>

</Customers>

</xmlmessage>

 

Example 1:

 

 ! Sets the pointer to <xmlmessage><Customers>

 Xml:GotoTop()

 If Xml:FindNextNode('XMLMessage','Customers')

         Message('Node Not Found!!','Error')

 End

 

Example 2:

 

 ! Sets the pointer to <xmlmessage><Customers><Customer>

 Xml:GotoTop()

 If Xml:FindNextNode('Customers','Customer')

         Message('Node Not Found!!','Error')

 End

 

Example 3:

 

 ! Sets the pointer to <xmlmessage><Customers><Customer>

 Xml:GotoTop()

 If Xml:FindNextNode('Customers','Customer')

         Message('Node Not Found!!','Error')

 ELSE

 

         ! Sets the pointer to <xmlmessage><Customers><Customer><personname>

         IF NOT XML:FindNextNode('PersonName')

 

                 ! Tries to find a <Customer> inside this parent and fails

                 IF XML:FindNextNode('Customer')                ! RETURNS ERROR

                         MESSAGE('No sub-customer found!')

                 END

         END

 END
