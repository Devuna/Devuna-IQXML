Parser - Loading Queue at Once

When you are processing a simple queue, (one that does not contain multiple children or you are not interested in the child data), you can use the "True" parameter on the XML:LoadQueue() function.  Once you do this, you can then process the entire queue all at once.

 

Take for instance the following xml data:

 

<xmlmessage>

<Customers>

<Customer>

  <PersonName>

 <GivenName>Robert</GivenName><Surname>Paresi</Surname></PersonName>

  <Address>

      <AddressLine>111 Main Street</AddressLine>

      <AddressLine>PO Box 12</AddressLine>

  </Address>

  <CityName>Hartford</CityName>

  <PostalCode>06331</PostalCode><StateProv>CT</StateProv>

</Customer>

<Customer>

  <PersonName>

 <GivenName>Robert</GivenName><Surname>Jones</Surname></PersonName>

  <Address>

      <AddressLine>1 Dunninghurst Ave</AddressLine>

  </Address>

  <CityName>Elmhurst</CityName>

  <PostalCode>12122</PostalCode><StateProv>NY</StateProv>

</Customer>

</Customers>

</xmlmessage>

 

You can load the entire queue easily by using the following method:

 

If ~Xml:LoadFromFile('testing.xml',False)

 If ~xml:FindNextNode('Customers','Customer')

 

         ! Load all at once (siblings)        

         ! -----------------------------------

           TotalRecords = LoadQueue(MyQueue,True,True)          

 

 End

End

 

The queue would contain two records.   Even if there were children records, and you were not concerned with them, the system would be able to process the entire document successfully.  It is important to set the "Next Node" before issuing the LoadQueue(queue, True, True) statement so the function knows when to create each queue record – such as who is the actual parent.
