General Use

These XML functions are designed to make working with XML documents easy for both ABC and Clarion template chains as well as hand-coding.  No classes are used to attach to your application, however, only one DLL is needed with your release plus DOS Driver.  

 

An example application is included with the installation of this development tool where you can try out the functions.  The purpose of these functions allows the user to easily add XML functionality to their Clarion application with some added and easy-to-use support such as ability to:
 

·Prime and cascade fields before adding queue records
·Debug your XML documents and step through with an interactive node-pointer
·Display your local Clarion queue interactively after filling it
·Easily position the node pointer to any node using one statement without having to look at every line.

 

Because of the flexibility of the node pointer in iQ-XML, if one customer were to send you data in a 'deeper' node than another customer, the same code will work seamlessly.
