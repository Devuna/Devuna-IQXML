XML:AddFromQueue

Parameters:

QUEUE

Local Queue to write out to XML file

STRING

Based tag for each record

[<BYTE>

True = Write out as Attribute Based

<BYTE>]

True = Auto-Close Group Tag (Attribute Based Only)

<BYTE>

True = Write all field names out as LowerCase

<BYTE>

True = Ignore any attributes which have a blank/empty value



Return Type:

Does not return a value

 

Reads a queue and writes out to an XML document all fields.  Field prefixes are ignored and stripped in field names. This function supports both Groups and Dimmed Fields to write out an XML document using multiple sibling and child records with different parent tags.

 

Fields are written in all uppercase, as that is how the variable is stored in memory in Clarion.  However, if you use a named attribute, then you are able to control the case (or the actual tag name) being written.  This also allows you to have one name in the QUEUE, but use a different name that is actually used as the tag name.

 

Ignore any attributes which have a blank/empty value:

Some schemas do not allow blank attributes and will return an error.  You can please True in the 6th parameters to not write out attributes of a blank value.  This would only be valid if the "Write out as Attribute Based" is set to true as well.

 

If DIM fields are used, the same amount of dimensions must exist within the group, otherwise, only the first value will be written out.

 

NOTE: DIMMED Field and Multi-Group Processing is not valid with Clarion 5.5 Version of iQ-XML

 

Example:

 

VALID

MyQueue


QUEUE

FirstName

STRING(20),NAME('FirstName')

LastName

STRING(20),NAME('LastName')

Invoice

GROUP()

InvoiceAmt

STRING(20),DIM(100)

InvoiceDate

STRING(20),DIM(100)

InvoiceNo

STRING(20),DIM(100)

END

END

 

INVALID

INVOICE

GROUP()

INVOICEAMT

STRING(20),DIM(100)

INVOICEDATE

STRING(20)

INVOICENO

STRING(20),DIM(100)

 

Using the above Valid example, the following would be created:

 

<Customer  Firstname="Robert" Lastname="Paresi">

<INVOICE>

<INVOICEAMT>10.00</INVOICEAMT>

<INVOICEDATE>2005-01-01</INVOICEDATE>

<INVOICENO>1</INVOICENO>

</INVOICE>

<INVOICE>

<INVOICEAMT>20.00</INVOICEAMT>

<INVOICEDATE>2005-01-02</INVOICEDATE>

<INVOICENO>2</INVOICENO>

</INVOICE>

</Customer>

 

If the Group has only ONE Dimensioned element, it will be written as multiple nodes inside one node.
