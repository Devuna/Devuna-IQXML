XML:SetProgressWindow

Parameters:

LONG

Number of Bytes to consider opening a progress window.
 
-1 = Never show window
 
 

<STRING>

String to display in the progress window.   (Blank displays "Loading Data ...")
 

-or-
 

Valid Clarion Target of a Progress control to update on your local window (Current Target)



Return Type:

Does not return value

 

If this is set, the XML:LoadFromString and XML:LoadFromFile functions will open a progress window if the number of bytes being processed is equal to or greater than the first parameter.  If the parameter is set to 0 (zero), the progress window will never be displayed.

 

By default, this window appears all the time and if 0 is called as a parameter.

 

The value set is available for the entire length of the thread and only that thread.

 

Instead of sending a message, you can send a Target of a ?Progress control on your local window that will be updated.

 

 

Example:

 

XML:SetProgressWindow(10000,'Please Wait while we do this!')

IF XML:LoadFromFile('myfile.xml')

 

END

 

! Updates the progress control on your local active window

! --------------------------------------------------------

XML:SetProgressWindow(1,?MyProgressControl)

IF XML:LoadFromFile('myfile.xml')

 

END

 

! Shows a Progress Window with "Loading Data ... Please Wait"

! ------------------------------------------------------------

XML:SetProgressWindow(1)

IF XML:LoadFromFile('myfile.xml')

 

END

 

! Hides a Progress Window

! ------------------------

XML:SetProgressWindow(-1)

IF XML:LoadFromFile('myfile.xml')

 

END
