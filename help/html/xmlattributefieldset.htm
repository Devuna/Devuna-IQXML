XML:AttributeFieldSet

Parameters:

STRING

Field name in LocalQueue

<BYTE Required>

IQXML:NONE (0)

IQXML:REQUIRED (1)

IQXML:OPTIONAL (2)
IQXML:FORCEWRITE (100)

 

<STRING Picture>

A valid Clarion picture to format the string before being placed in the Queue field.

 



Return Type:

Does not return a value

 

Further defines a field for the XML:LoadQueue() function.  If you have added the global template, you can use the Equates.  These values are not cleared until the XML:AttributeFieldClear() function is called or the XML:Free statement is called.

 

Required:

 

IQXML:REQUIRED

If this field is blank, the record will not be added to the passed Queue, even if other fields contain values.

 

IQXML:OPTIONAL

If this field has a value, a record will not be added to the passed Queue unless another non-optional field is populated.

 

IQXML:FORCEWRITE

A record will be inserted into the queue immediately when this field or attribute is reached, regardless of whether or not all, some or none of the fields have data in them.   Used when you know that all data has been collected for this one record regardless of the parent node being closed or ended.  Even if the data itself is blank, a queue record will be written.
 

 

String values in an XML document which contain "0" or "false" or "0.00" will be considered a 'filled' field.  Only BLANK fields are considered unfilled, ones that are actually blank in the XML document.

 

 

 <Name Firstname="John" Lastname="Smith">

 <Profiles>

    <Profile Type="1">

         <Phone>555-1212</Phone>

         <Email>me@my.com</Email>

        </Profile>

  <Profile Type="2">

         <Phone></Phone>

         <Email></Email>

        </Profile>

    <Profile Type="3">

         <Phone></Phone>

         <Email>lester@nester.com</Email>

        </Profile>

 </Profiles>

 </Name>

 

MyQueue

QUEUE

Profile

string(20)

Phone

string(20)

Email

string(20)

END

 

 XML:AttributeFieldSet('Profile',IQXML:OPTIONAL)

 IF NOT XML:FindNextNode('Profiles','Profile')

         XML:LoadQueue(MyQueue,True,True)

 END

 

In the above example, only two record will be added to the Queue.  If the AttributeFieldSet was not set, then all three records would end up in the Queue.

 

 

 XML:AttributeFieldSet('Phone',IQXML:REQUIRED)

 IF NOT XML:FindNextNode('Profiles','Profile')

         XML:LoadQueue(MyQueue,True,True)

 END

 

In the above example, only one record (the first one) will be added to the Queue.  If the AttributeFieldSet was not set, then all three records would end up in the Queue.

 

 

 

 <Name Firstname="John" Lastname="Smith">

 <Notes>

    <Note>This is a note!</Note>

    <Note>This is a note!</Note>

    <Note></Note>

    <Note>This is a note!</Note>

 </Notes>

 </Name>

 

 XML:AttributeFieldSet('Note',IQXML:FORCEWRITE)

 IF NOT XML:FindNextNode('Name')

         XML:LoadQueue(MyQueue,True,True)

 END

 

In the above example, four records will be written out to the queue including the blank line, even though the sibling pointer points to "Name".

 

 

 XML:AttributeFieldSet('Note',IQXML:FORCEWRITE + IQXML:REQUIRED)

 IF NOT XML:FindNextNode('Name')

         XML:LoadQueue(MyQueue,True,True)

 END

 

In the above example, three records will be written out to the queue excluding the blank line, even though the sibling pointer points to "Name".

 

 

Here is an example of dynamically figuring out when to write out the record because there may be multiple-versions of the XML, and you are not sure which field is the last field in the group.   You may need this if you add additional fields at a later point and the FORCEWRITE may be different depending on how the XML file looks.

 

In the following example, Version 1 of the XML has a ending field of "ThatField" but Version 2 of the XML has a ending field of "ThisField".   First, we save the pointer state, then we look to see what version of the XML we are going to load, reset the state back to the original pointer and position, and then load the queue.

 

IF ~XML:FindNextNode('TheData')

   i = XML:SaveState()

   IF ~xml:FindNextNode('ThisField')

     XML:AttributeFieldSet('ThisField',IQXML:FORCEWRITE)

   else

     XML:AttributeFieldSet('ThatField',IQXML:FORCEWRITE)

   end

   If XML:RestoreState(i).

   XML:FreeState(i)

   if XML:LoadQueue(DynamicQueue,True,True).

   XML:AttributeFieldClear()

ELSE

   Do ShowXMLError

END
