XML:SetBoundry

Parameters:

STRING

Field not to include or read past



Return Type:

NONE

 

Ability to call this function once to indicate not to continue if this field is encountered.  (first time being encountered)

 

Calling this function as blank or calling XML:FREE() will clear this value.

 

Setting a boundary affects the following functions:

 

·XML:FindNextNode
·XML:FindNextContent
·XML:FindAnyNode
·XML:LoadQueue

 

 

The purpose of this function is to set a boundry to make the above functions more efficient is very large documents.  For example, if you are reading an XML document where the top half may have a configuration of taxes and the bottom half has ten thousand invoices, and you are only interested in loading the taxes into a queue - you can set a boundry of the first element for invoices.  This will cause the LoadQueue (if you were not within a Child Group) to exit out before it read to the very end.

 

If it important to know that XML:LoadQueue is already efficient in terms of only reading within a Child group that you have found.  But, for example, let's say you do not know if the document contains a group called Taxes - and you know that if it did, it would always appear before the Invoices - you could do something like:

 

xml:SetBoundry('Invoices')

if not xml:FindNextNode('Taxes')

 do LoadTaxes

end

xml:SetBoundry('')        ! clear the boundry

 

xml:gotoTop()

Do LoadInvoices

 

 

This example would at least make sure that if it found invoices, it wouldn't look anymore for Taxes.   If the document were under 100K, it would not make a difference, but if the document were say 100MB, it would make the functions run much faster.   Note that you must know that the XSD/XML Design from the provider would always have <Taxes> higher than <Invoices>

 

ie:

 

<InvoiceFeed>

<Taxes>

<!-- these are the agreed tax settings -->

</Taxes>

<Invoices>

<!-- These are the many invoices -->

</Invoices>

</InvoiceFeed>

 

 

 

1.<xmlmessage>
2.<Customers>
3. <Customer>
 

       xml:SetBoundry('Address')

 if not xml:FindNextNode('AddressLine')

         ! this will never happen

 else

         message('The node was not found because a boundary was set')

 end

 

 or the following will not make the LoadQueue function read anything past Invoices.

 

 xml:SetBoundry('Invoices')

 XML:GotoTop()

 Recs = xml:LoadQueue(Customers,True,True)

 xml:SetBoundry('')
