Parser - Using PrimaryFieldCascade

If you are processing an XML document with both Child and Sibling parameters and/or you need to attach records from one queue to another using some kind of primary key, you can set the value in the queue before calling the XML:LoadQueue() function.

 

Take for instance the following xml data:

 

<xmlmessage>

<Customers>

<Customer CustomerName="Smith Travel" CustomerID=123>

  <Invoices>

    <Invoice Invoiceno="1" InvoiceDate="2005-01-01" InvoiceAmt="12.12"/>

    <Invoice Invoiceno="12" InvoiceDate="2005-03-02" InvoiceAmt="9.09"/>

    <Invoice Invoiceno="144" InvoiceDate="2005-04-02" InvoiceAmt="1000.00"/>

  </Invoices>

</Customer>

</xmlmessage>

 

The following code will allow you to process this XML document process the data in to two queues, but link the two sets of data together with a primary key, using field "CustomerID".

 

 

CustomerQueue

QUEUE,PRE(CUS)

CustomerName

string(20)

CustomerID

string(20)

End

InvoicesQueue

QUEUE,PRE(INV)

CustomerID

string(20)

InvoiceNo

string(20)

InvoiceDate

string(20)

InvoiceAmt

string(20)

End

IRecs

SHORT

SaveState

SHORT

 

CODE

 

If NOT Xml:LoadFromFile('testing.xml')

 

 If NOT xml:FindNextNode('Customers','Customer')

 

 LOOP

         XML:AutoRootSet()

 

         IRecs = XML:LoadQueue(CustomerQueue,False,False)

         If Irecs <> 0

                 Get(CustomerQueue,records(CustomerQueue))

                 Xml:PrimaryFieldCascade('CustomerID',cus:CustomerID)

 

                 If NOT xml:FindNextNode('Invoices','Invoice')

                         IRecs = XML:LoadQueue(InvoicesQueue,True,True)

                 end

 

                 XML:PrimaryFieldClear()

         End

 

         IF XML:GotoSibling() THEN BREAK.

 

  END

 End

End

 

 

Example 2:

 

Here is another example of filling two queues of information.  Notice the parent stay information contains header (summary) information, and the child records contain the detail.  You can use the XML:PrimaryFieldCascade to attach both records together, especially in this case where both the Header and Detail attributes use the same Field Names.

 

<Reservation>

<StayInformation Firstname="Robert" Lastname="Paresi" ID="1111">

<Stay StartDate="2005-01-01" EndDate="2005-01-30" Total="1950.00">

<StayDetail>

       <RoomRate StartDate="2005-01-01" EndDate="2005-01-15" Rate="50.00" />

       <RoomRate StartDate="2005-01-16" EndDate="2005-01-30" Rate="80.00" />

</StayDetail>

</Stay>

<Stay StartDate="2005-03-01" EndDate="2005-03-30" Total="2835.00">

<StayDetail>

       <RoomRate StartDate="2005-03-01" EndDate="2005-03-15" Rate="90.00" />

       <RoomRate StartDate="2005-03-16" EndDate="2005-03-30" Rate="99.00" />

</StayDetail>

</Stay>

</StayInformation>

</Reservation>

 

HRateQueue

QUEUE,PRE(H1)

Sysid

STRING(20)

StartDate

string(20)

EndDate

string(20)

Total

string(20)

End

DRateQueue

QUEUE,PRE(D1)

Sysid

STRING(20)

StartDate

string(20)

EndDate

string(20)

Rate

string(20)

End


NDX

SHORT

I

SHORT

 

IF ~XML:FindNextNode('StayInformation','Stay')

       LOOP I = 1 TO 255

          XML:PrimaryFieldCascade('SysId',format(I,@n03))

          NDX = XML:LoadQueue(HRateQueue,False,False)

          IF NDX

             SaveState = XML:SaveState()

             If ~XML:FindNextNode('StayDetail','RoomRate')

                 NDX = XML:LoadQueue(DRateQueue,True,True)

             End

             if ~XML:RestoreState(SaveState)

                XML:FreeState(SaveState)

             END

           END

           XML:PrimaryFieldClear()

           if XML:GotoSibling() then break.  ! <StayInformation><Stay>

         END

      end

 xml:free()

end
