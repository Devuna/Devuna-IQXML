XML:PrimaryFieldCascade

Parameters:

STRING

Field Name to Prime in Queue

 

STRING

Default Value for Field Name



Return Type:

Does not return a value

 

Used to tell the XML:LoadQueue function to prime these fields with a certain value when adding new 'child' and 'sibling' entries to the queue. These values are maintained until XML:PrimaryFieldClear is called.

 

There are two functions which Prime fields in your queue and this is one of them.  The other is "XML:PrimaryKeyCascade"

 

PrimaryFieldCascade is used to push the value in to the field while PrimaryKeyCascade is used to retain the value after the previous inserted record.  Usually, if you are manually processing an XML section, you'll use PrimaryFieldCascade where-as if you are processing the entire document, you'd use the PrimaryKeyCascade.

 

See also XML:PrimaryFieldClear XML:PrimaryKeyCascade

 

 

Example:

 

! Load a Header of Customers and Detail of Invoices attaching all

! the right invoices to the proper customers

 

If NOT XML:FindNextNode('Customers')        ! find first customer record

 LOOP

               AddedRecords = XML:LoadQueue(HeaderQueue,False,False)

         If AddedRecords

                 Get(HeaderQueue,records(HeaderQueue))

                 Xml:PrimaryFieldCascade('CustomerID',H1:CustomerID)

                  AddedRecords = XML:LoadQueue(DetailQueue,True,True)

                 XML:PrimaryFieldClear()

         END

         If XML:GotoSibling() then break.        ! Break when done all records

 END

END
