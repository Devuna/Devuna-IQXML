XML:BinaryFieldSet

Parameters:

STRING

Field name


<STRING>

Directory where to place binary encoded objects



Return Type:

Does not return a value

 

Important Note:  Unlike other Set functions, this must be called prior to XML:LoadFromFile or XML:LoadFromString.

 

Defines that this field is an encoded BASE64 field and should be decoded.   Once the field is decoded, it is placed in a temporary object in the current directory or the directory specified in the second parameter.  Currently, the only supported encoding is Base64.

 

You retrieve the pointer/name of the object, the field in the XML:LoadFromQueue, XML:ReadNextRecord, XML:ReadPreviousRecord will contain the pointer/name of the object rather than the value.

 

You must move the object or process the object before XML:Free is called, otherwise the temporary object is removed.  XML:Free will remove all objects created. If the object no longer exists during XML:Free, no error will be displayed so it is fine to simply move the object or rename it should you need to and it.

 

See also XML:BinaryFieldClear

 

Example:        

 

 

FilePointer        CSTRING(64)

 

myqueue        queue()

Signature        string(64)

Account        string(10)

Name                string(50)

 end

 

 CODE

 

 XML:BinaryFieldSet('Signature')

 

 IF NOT XML:LoadQueue(MyQueue)                          

         if NOT XML:FindNextNode('Signature')

                       if NOT XML:ReadCurrentData(FilePointer)

                         COPY(FilePointer,'CustomerSignature.gif')

                 END

         END        

 END

 

 XML:FREE()
