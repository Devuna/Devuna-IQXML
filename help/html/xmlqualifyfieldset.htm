XML:QualifyFieldSet

Parameters:

STRING

Field name to Qualify to Parent

<BYTE>

1 = Qualify to GrandparentParent
2 = Qualify to GreatGrandparentParent




Return Type:

 

Does not return a value

 

Instructs the system to qualify this field name with its parent before matching fields during the XML:LoadQueue() function. You can call this function as many times as needed for as many unique fields you need to qualify.

 

If you use the XML:LoadQueue function, you may come across a field name being used more than once and you are only concerned with the data that corresponds to one of them or that the field will be used more than once in your queue to mean different things. Common named fields used through-out an XML document may cause you an issue when using the XML:LoadQueue statement such as "Code", "Data", "Number", where the same field name could be used in multiple places, but mean different things.

 

Qualified field names are concatenated in the form of "ParentAttribute".

Double Qualified field names are concatenated in the form of "GrandparentParentAttribute"

Triple Qualified field names are concatenated in the form of "GreatGrandparentParentAttribute"

 

 

The maximum length of the field, both element and attribute combined, is 128 characters.

 

<CompanyName Code="1213" Data="123"> qualifies to CompanyNameCode and CompanyNameData

 

 

NOTE: If the XML:LoadQueue function is set to ignore prefixes/name spaces, then the QualifyFieldSet should not include the prefixes as well so there is a match.   This would be when you use the "2" option for Include field prefixes.

 

 

See also XML:QualifyFieldClear

 

 

Example 1:

 

Take for example the portion of this XML document:

 

 <Data>

 <Name Firstname="John" Lastname="Smith">

 <Phones>

    <Home Number="555-1212">

         <Type>Cell</Type>

    <Work Number="555-1212">

         <Type>Fax</Type>

  <Mobile Number="555-9999">

         <Type>Cell</Type>

 </Phones>

 </Name>

    </Data>

 

Notice each has the field name "Number", however, you only want to bring the Home and Work numbers into the Queue.   The following code will qualify the XML:LoadQueue operation to be more specific when working with such a field name before matching.  If you did not qualify the field, you would end up with three records in your queue.

 

 

MyQueue

QUEUE

Firstname

string(20)

LastName

string(20)

HomeNumber

string(20)

PhonesHomeType

string(20)

WorkNumber

string(20)

PhonesWorkType

string(20)

END

 

 IF NOT XML:FindNextNode('Name')

               XML:QualifyFieldSet('Number')

         XML:QualifyFieldSet('Type',True)        ! Qualify <Parent><Parent><element>

         XML:LoadQueue('Names',True,True)

         XML:QualifyFieldClear('Number')

 END

 

 

 

 

 

Example 2:

 

Take for example the portion of this XML document:

 

 <Name Firstname="John" Lastname="Smith">

 <Employee>

    <BadgeInfo>

         <Code>12344</Code>

    </BadgeInfo>

 <ProgramSkills Code="Yes" Design="Hell no" Document="Terrible"/>

 </Employee>

 </Name>

 

 

Notice each there are two "Code" fields, however, we are only concerned with the Programming Skills type.

 

 

MyQueue

QUEUE

Firstname

string(20)

LastName

string(20)

BadgeInfoCode

string(20)

ProgramSkillsCode

string(20)

END

 

 IF NOT XML:FindNextNode('Name')

               XML:QualifyFieldSet('Code')

         XML:LoadQueue('Names',True,True)

               XML:QualifyFieldClear()

 END
