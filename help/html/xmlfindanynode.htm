XML:FindAnyNode

Parameters:

STRING

Element Name

<STRING>

Optional Element Name 2

<STRING>

Optional Element Name 3

<STRING>

Optional Element Name 4

<STRING>

Optional Element Name 5






Return Type:

SHORT

Which Element was Found (1 through 5)
0 = End of File

-1 = Not within the current node pointer

 

Used to find the first tag occurrence of any of the 1 through 5 elements. When the first one found, the current Node Pointer will be set to that record.  Using this function will always start at the current cursor location.

 

This works like the XML:FindNextNode, except that it will find ANY of the (up to) five single elements and return which one it found.   Could be used if you are looking at an incoming document to see if it is any of a certain documents be sent.  ie:  Is it a Purchase Order, an Inventory Item update, a Customer Update, or a Billing Notification.

 

In this case, XML:FindAnyNode should be used instead of multiple XML:FindNextNode as it is more efficient:

 

Example:

 

  if not xml:FindNextNode('Invoices')

 Type = 'I'

  elsif not xml:FindNextNode('Customers')

 Type = 'C'

  elsif not xml:FindNextNode('PurchaseOrders')

 Type = 'P'

  elsif not xml:FindNextNode('Billing')

 Type = 'B'

  else

 Type = ''

  end

 

Using xml:FindAnyNode, it is more efficient/faster:

 

execute xml:FindAnyNode('Invoices','Customers','PurchaseOrders','Billing')

Type = 'I'

Type = 'C'

Type = 'P'

Type = 'B'

else

 Type = ''

end

 

 

It's very helpful to use this if you are receiving multiple documents, but you want to see which one it is.

 

 

WhichOne   SHORT

 

CODE

WhichOne = xml:FindAnyNode('Invoices','Customers','PurchaseOrders','Billing')

 

 

WhichOne would return as 2 because it found Customers
