XML:SetEncoding

Parameters:

STRING

Encoding type such as UTF-8

<BYTE>

True = Prefix XMLCreateFile with needed header




Return Type:

Does not return value

 

If no Encoding value is located in the header of the XML file, then iQ-XML assumes ISO-8859-1 (Default), which accepts all ASCII characters (Latin and extended ASCII) - All characters are coded in single eight-bit code and unconverted.  There may be times when you want to force UTF-8 or some other encoding when the party who created the XML file did not set the encoding properly.

 

Before issuing the XML:LoadFromFile or XML:LoadFromString function, you can force the encoding type.  If you force the encoding type, if the XML file has an encoding value, it will be ignored.

 

You must call XML:SetEncoding prior to every XML:LoadFromFile or XML:LoadFromString function.  This value is cleared after each load.

 

xml:SetEncoding('UTF-8')

If ~Xml:LoadFromFile('testing.xml',False)

 

xml:SetEncoding('UTF-8',True)

IF NOT XML:CreateXMLFile('c:\outbound\myfile.xml','1.0','UTF-8',True)

 

Note: Most/All XML Parsers will not read the first three bytes or use them (Web Service) so it would not be needed...but some standalone notepads/browsers would require this as it doesn't use the XML specification to know UTF but the actual header bytes.  (EF BB BF)   It all depends on if you are going to be opening the created XML file in a browser/notepad, or are you actually sending it to XML parser who will read the "encoding=" portion rather than the header file bytes.

 

 

 

For writing a file with an Encoding override, the file must be created first and then the encoding being overridden.

 

If ~Xml:CreateXMLFile('testing.xml','1.0','UTF-8')

xml:SetEncoding('Windows-1251')
