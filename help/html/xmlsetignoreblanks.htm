XML:SetIgnoreEmpty

Parameters:

BYTE

Ignore Empty Attributes (True/False)

BYTE

Ignore Empty Elements (True/False)



Return Type:

Does not return a value

 

Instructs the writer to ignore empty attributes and elements.   If the value is empty, the writer will ignore the write automatically as if there was no code to write it.  It is common in XML not to write blank elements and attributes as most XSD definitions do not allow it. A minimum length of string in most attributes is 1 and not 0.  Normally, the reader program on the other end assumes that missing items are considered blank.

 

 

Example:

 
 FirstName = 'John'

 LastName = 'Boy'

 Street = ''

 City = ''

 State = ''

 ZIPCode = ''

 

 XML:SetIgnoreEmpty(True,True)

 XML:CreateParent('Person');AddParent()

 

 XML:CreateParent('Name')

 XML:CreateAttribute('Firstname',FirstName)

 XML:CreateAttribute('Lastname',LastName)

 XML:AddParent(True)

 

 xml:AddElement('Street',Street)

 xml:AddElement(City',City)

 xml:AddElement('State',State)

 xml:AddElement('PostalCode',ZIPCode)

 

 XML:CloseParent()

 

 

<Person>

<Name Firstname="John" Lastname="Boy"/>

</Person>
